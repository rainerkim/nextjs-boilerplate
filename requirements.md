# 개인 다이어리 웹 애플리케이션 요구사항 명세서

## 1. 프로젝트 개요
개인이 일상을 기록하고 관리할 수 있는 웹 애플리케이션 개발을 목표로 합니다. 사용자는 날짜별로 일기를 작성, 조회, 수정, 삭제할 수 있으며, 날짜에 따라 고유한 색상 테마가 적용되는 감성적인 UI를 제공합니다.

## 2. 기능 요구사항

### 2.1 일기 쓰기 (Create)
- **날짜 선택**: 사용자는 일기를 작성할 날짜를 선택할 수 있습니다.
- **내용 입력**: 제목과 텍스트 본문을 입력하여 일상을 기록합니다.

### 2.2 일기 목록 조회 (Read - List)
- 저장된 모든 일기를 리스트 형태로 확인할 수 있습니다.
- 각 리스트 아이템은 기본적인 정보(날짜, 제목 등)를 표시합니다.

### 2.3 일기 상세 조회 (Read - Detail)
- 리스트에서 특정 일기를 클릭하면 상세 페이지로 이동합니다.
- 상세 페이지에서는 작성한 일기의 전문을 확인할 수 있습니다.

### 2.4 일기 수정 및 삭제 (Update / Delete)
- **수정**: 상세 페이지에서 기존에 작성된 일기의 내용을 수정할 수 있습니다.
- **삭제**: 상세 페이지에서 해당 일기를 삭제할 수 있습니다.

## 3. UI/UX 디자인 요구사항
- **동적 컬러 테마**:
    - 날짜 포맷인 `yyyyMMdd` 값을 기반으로 배경 컬러와 텍스트 컬러를 자동으로 생성합니다.
    - 생성된 배경색과 텍스트 색상은 서로 시각적으로 어울리며 가독성이 좋아야 합니다.

## 4. 기술 및 데이터 요구사항
- **데이터 저장소**: 별도의 데이터베이스 시스템을 사용하지 않고, 파일 시스템을 이용합니다.
- **파일 포맷**: 모든 일기 데이터는 **TSV (Tab-Separated Values)** 포맷의 파일로 저장합니다.
